# МережаЗв'язків • Теорія Графів

Вебдодаток для інтерактивної візуалізації та аналізу топології соціальних мереж. Проєкт реалізує динамічне відображення зв'язків між користувачами на основі алгоритмів теорії графів та пропонує інтелектуальну систему рекомендацій друзів за допомогою методів машинного навчання (косинусна подібність).

---

## 🚀 Основний функціонал

* **Динамічна візуалізація графа (D3.js Force Simulation):** Вершини графа представляють користувачів, а ребра — соціальні зв'язки між ними. Реалізовано симуляцію фізичних сил (відштовхування, зіткнення, центрування) з підтримкою Drag-and-Drop для перетягування вершин.
* **Інтелектуальні рекомендації (ML Feature):** Система використовує алгоритм **Cosine Similarity (Косинусна подібність)** для порівняння текстових векторів інтересів користувачів. На основі цього додаток рекомендує найбільш релевантних потенційних друзів із відображенням відсотка збігу.
* **Динамічна вага ребер:** Товщина лінії зв'язку (ребра) динамічно змінюється залежно від кількості спільних інтересів між двома користувачами.
* **Пошук центральних хабів (Degree Centrality):** Функція «Знайти Хаб» автоматично вираховує вершину з найбільшим степенем (найбільшою кількістю зв'язків), фокусує на ній камеру та підсвічує її в інтерфейсі.
* **Режим інтерактивного редагування (Add/Delete Mode):** * За допомогою плаваючої кнопки (FAB) можна увімкнути режим редагування зв'язків, де створення або розірвання ребра відбувається в один клік по вершинах.
* Реалізовано кастомне контекстне меню (правий клік миші) на ребрах для швидкого видалення зв'язку.


* **Аналітика інтересів (Interest Cloud):** Окрема вкладка збирає всі інтереси користувачів, сортує їх за популярністю та відображає статистику в реальному часі.
* **Сучасний UI/UX інтерфейс:** Дизайн виконано в ультрасучасному стилі **Glassmorphism (скляний морфізм)** із використанням темної неонової палітри кольорів, плавних анімацій, системи спливаючих сповіщень (Toasts) та адаптивних вкладок.

## 🛠 Технологічний стек

* **Frontend:** HTML5, CSS3 (Custom Variables, Flexbox, Grid, Glassmorphic панели)
* **Шрифти та іконки:** Google Fonts (Inter), FontAwesome v6.6.0

## 📁 Структура проєкту

* `index.html` — Головна сторінка додатка. Містить розмітку сайдбара, шапки, робочої області графа, карток профілів, рекомендацій та хмари інтересів.
* `styles.css` — Стилі інтерфейсу користувача. Описує Glassmorphism-ефекти, анімацію пульсації FAB-кнопки, затемнення неактивних елементів графа та адаптивну сітку.
* `data.js` — Модуль керування даними. Відповідає за автогенерацію профілів (28 унікальних імен та базований пул із 32 інтересів), реалізацію математичного алгоритму `calculateCosineSimilarity` та функцій додавання/видалення ребер.
* `graph.js` — Ядро візуалізації D3.js. Налаштовує фізичні сили (`forceManyBody`, `forceCollide`), рендерить SVG-елементи, обробляє події миші, контекстного меню та керує анімацією пошуку/підсвічування.
* `ui.js` — Логіка керування інтерфейсом користувача (UI). Забезпечує перемикання вкладок, роботу глобального пошуку за ім'ям, роботу панелей профілю та оновлення статистики в реальному часі.

## 🧮 Використані алгоритми

### 1. Косинусна подібність (Cosine Similarity)

Для порівняння профілів інтереси користувачів представляються у вигляді векторів. Алгоритм рахує перетин множин інтересів та ділить його на геометричне середнє потужностей цих множин:


$$\text{Similarity}(p_1, p_2) = \frac{|I_1 \cap I_2|}{\sqrt{|I_1|} \cdot \sqrt{|I_2|}}$$

### 2. Степінь центральності (Degree Centrality)

Для знаходження головного «Хабу» мережі додаток аналізує масив ребер (`edges`) і знаходить користувача з максимальною кількістю інцидентних ребер:


$$\text{Degree}(v) = \sum_{u \in V} e(v, u)$$

## 🔮 Перспективи розвитку (Roadmap)

Відповідно до початкових вимог та специфікації проєкту, у майбутньому заплановано наступні покращення:

1. **Впровадження Графових нейронних мереж (Graph Neural Networks / GNN):** Для глибшого передбачення зв'язків та аналізу прихованих закономірностей у топології мережі.
2. **Інтеграція з WebXR:** Перенесення візуалізації графа у тривимірний простір доповненої (AR) та віртуальної (VR) реальності для інтерактивної взаємодії.
3. **Розширення до транспортних систем:** Адаптація алгоритмів аналізу топології мережі для оптимізації логістичних та транспортних маршрутів.

## 🛠 Як запустити проєкт

Проєкт не потребує додаткових інструментів збірки чи локальних серверів.

1. Склонуйте репозиторій або завантажте файли.
2. Переконайтеся, що всі 5 файлів (`index.html`, `styles.css`, `data.js`, `graph.js`, `ui.js`) знаходяться в одній папці.
3. Відкрийте файл `index.html` у будь-якому сучасному веббраузері.
